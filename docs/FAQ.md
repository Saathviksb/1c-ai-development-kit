# Часто задаваемые вопросы (FAQ)

## Общие вопросы

### Что такое 1C AI Development Kit?

Это комплексный набор инструментов для эффективной разработки на платформе 1С:Предприятие с использованием AI-ассистентов в Cursor IDE. Включает агентов, навыки, правила и MCP-серверы.

### Нужна ли подписка для использования?

Базовый набор (агенты, навыки, правила) — **бесплатный**. Некоторые MCP-серверы платные, но можно использовать и без них.

### Какие версии 1С поддерживаются?

1С:Предприятие 8.3 (все редакции: базовая, КОРП, УТ, ERP и т.д.)

### Работает ли с EDT (Eclipse Development Tools)?

Пока нет, но в планах. Сейчас поддерживается только Cursor IDE.

## Установка и настройка

### Как установить базовый набор?

```bash
git clone https://github.com/Jefest9988/1c-ai-development-kit.git
cd 1c-ai-development-kit

# Скопируйте в ваш проект
cp -r .cursor/* /path/to/your-project/.cursor/
```

См. [Руководство по установке](guides/installation.md)

### Нужно ли устанавливать все MCP-серверы?

Нет. Начните с бесплатных (`bsl-lsp-bridge`, `rlm-toolkit`), остальные добавьте по необходимости.

### Как получить доступ к платным MCP?

Платные MCP-серверы можно приобрести на [vibecoding1c.ru/mcp_server](https://vibecoding1c.ru/mcp_server)

**Стоимость:** 7000₽ за комплект из 8 MCP-серверов

**Что входит:**
1. HelpSearchServer — поиск по справке 1С (RAG)
2. Graph Metadata Search — графовый поиск по метаданным
3. CodeMetadataSearchServer — поиск по коду и метаданным
4. SSLSearchServer — справка по БСП
5. SyntaxCheckServer — проверка синтаксиса (BSL LS)
6. TemplatesSearchServer — библиотека шаблонов
7. 1CCodeChecker — проверка через 1С:Напарник
8. FormsServer — работа с формами

**Автор:** Олег Филиппов ([@comol_foa](https://t.me/comol_foa))  
**Сообщество:** [t.me/comol_it_does_matter](https://t.me/comol_it_does_matter)

### Можно ли использовать без MCP-серверов?

Да! Агенты, навыки и правила работают и без MCP. MCP-серверы — это дополнительные возможности.

## Использование

### Как запустить агента?

В Cursor IDE:
1. Откройте чат
2. Напишите задачу, например: "Проверь код модуля X"
3. AI автоматически выберет подходящего агента (`onec-code-reviewer`)

Или явно: "Используй агента onec-code-reviewer для проверки модуля X"

### Как использовать навык?

Навыки используются автоматически, когда AI определяет подходящую задачу. Или явно:

```
Пользователь: Используй навык 1c-forms/compile для создания формы
```

### Какой агент выбрать для задачи?

| Задача | Агент |
|--------|-------|
| Написать код | `onec-code-writer` |
| Проверить код | `onec-code-reviewer` |
| Спроектировать архитектуру | `onec-code-architect` |
| Понять как работает код | `onec-code-explorer` |
| Упростить код | `onec-code-simplifier` |
| Создать форму | `onec-form-generator` |
| Оптимизировать запрос | `onec-query-optimizer` |
| Создать тесты | `onec-test-generator` |

### Как сэкономить токены?

1. Используйте `rlm-toolkit` для сохранения контекста между чатами
2. Запускайте агентов параллельно (изолированный контекст)
3. Начинайте новый чат при достижении 100k токенов
4. Используйте Sonnet 4.5 для рутинных задач, Opus 4.6 только для критичных

См. [model-selection.mdc](.cursor/rules/model-selection.mdc)

## Проектные MCP

### Зачем нужны проектные MCP?

Они индексируют вашу конфигурацию и позволяют AI:
- Искать по метаданным и коду семантически
- Строить граф зависимостей
- Отвечать на вопросы о вашем проекте

### Сколько стоит создать проектный MCP?

**Облачная индексация:**
- Малая конфигурация (<10k строк): $0.10-0.20
- Средняя (10-50k): $0.50-1.00
- Большая (50-200k): $1.00-2.00

**Локальная индексация:** бесплатно, но требует времени (часы)

### Как часто нужно переиндексировать?

При изменении конфигурации. Можно настроить автоматическую переиндексацию через Git hook.

### Можно ли использовать без GPU?

Да, но медленнее. Рекомендуется GPU для локальной индексации.

## Проблемы

### Агенты не работают

**Проверьте:**
1. Cursor IDE версия 0.40+ (с поддержкой MCP)
2. Файлы агентов в `.cursor/agents/`
3. Перезапустите Cursor (Ctrl+Shift+P → "Reload Window")

### MCP-сервер не отвечает

**Проверьте:**
1. Сервер запущен: `curl http://localhost:PORT/health`
2. Порт в `.cursor/mcp.json` совпадает с портом сервера
3. Логи сервера

### Навыки не выполняются

**Проверьте:**
1. Файлы `SKILL.md` на месте
2. Права выполнения для скриптов (`.ps1`, `.bat`)
3. Попросите AI явно: "Используй навык X"

### AI не следует стандартам кодирования

**Проверьте:**
1. Файл `.cursor/rules/1c-coding-standards.mdc` на месте
2. Правила не переопределены в других файлах
3. Попросите AI явно: "Следуй стандартам из 1c-coding-standards.mdc"

### Ошибка "Token limit exceeded"

**Решение:**
1. Начните новый чат
2. Используйте `rlm-toolkit` для сохранения контекста
3. Запускайте агентов для изоляции контекста

## Разработка

### Как создать свой навык?

1. Изучите существующие навыки в `.cursor/skills/`
2. Создайте папку `your-skill/`
3. Создайте `SKILL.md` по [шаблону](templates/skill-template.md)
4. Добавьте скрипты в `scripts/`
5. Протестируйте

См. [Создание навыков](guides/custom-skills.md)

### Как создать своего агента?

1. Изучите существующих агентов в `.cursor/agents/`
2. Создайте файл `your-agent.md` с frontmatter
3. Опишите роль, ответственности, инструменты
4. Добавьте примеры использования

См. [Создание агентов](guides/custom-agents.md)

### Как добавить новое правило?

1. Создайте файл `.cursor/rules/your-rule.mdc`
2. Используйте YAML блоки для структуры
3. Добавьте примеры и критические правила
4. Перезапустите Cursor

См. [Создание правил](guides/custom-rules.md)

## Безопасность

### Безопасно ли отправлять код в MCP?

**Бесплатные MCP (локальные):** Да, код не покидает вашу сеть

**Платные MCP (облачные):** Зависит от провайдера. Уточните политику конфиденциальности.

**Проектные MCP:** Вы контролируете, где хранятся данные (локально или в облаке)

### Как защитить credentials?

1. Добавьте в `.gitignore`: `*.env`, `.1c-devbase.bat`
2. Используйте переменные окружения
3. Не коммитьте файлы с паролями

### Можно ли использовать в закрытых проектах?

Да, лицензия MIT позволяет.
