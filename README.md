# 1C AI Development Kit

> Комплексный набор инструментов для разработки на платформе 1С:Предприятие с использованием AI-ассистентов в Cursor IDE

## 🎯 Что это?

**1C AI Development Kit** — это готовая экосистема для эффективной разработки на 1С с помощью AI. Включает:

- 🤖 **12 специализированных AI-агентов** для разных задач (код-ревью, генерация форм, оптимизация запросов и т.д.)
- 📚 **15+ навыков (skills)** для автоматизации рутинных операций
- 🔧 **9 правил (rules)** для контроля качества и стандартов кодирования
- 🌐 **8 MCP-серверов** для интеграции с документацией, валидацией кода и семантическим поиском
- 📖 **Полная документация** с примерами и best practices

## ⚡ Quick Start

### Требования

- [Cursor IDE](https://cursor.sh/) (версия с поддержкой MCP)
- 1С:Предприятие 8.3 (любая редакция)
- PowerShell 5.1+ (для Windows)
- Git

### Установка

1. **Клонируйте репозиторий:**

```bash
git clone https://gitea.yourdomain.com/yourname/1c-ai-development-kit.git
cd 1c-ai-development-kit
```

2. **Скопируйте файлы в ваш проект:**

```bash
# Агенты
cp -r .cursor/agents/* your-project/.cursor/agents/

# Правила
cp -r .cursor/rules/* your-project/.cursor/rules/

# Навыки
cp -r .cursor/skills/* your-project/.cursor/skills/
```

3. **Настройте MCP-серверы** (см. [docs/guides/mcp-setup.md](docs/guides/mcp-setup.md))

4. **Готово!** Откройте проект в Cursor и начните работу

## 📦 Что входит в комплект?

### AI-Агенты (.cursor/agents/)

| Агент | Назначение | Модель |
|-------|-----------|--------|
| `onec-code-writer` | Написание и модификация 1С кода | Sonnet 4.5 |
| `onec-code-reviewer` | Код-ревью с проверкой стандартов | Sonnet 4.5 / Opus 4.6 |
| `onec-code-architect` | Проектирование архитектуры | Opus 4.6 |
| `onec-code-explorer` | Анализ кодовой базы | Sonnet 4.5 |
| `onec-code-simplifier` | Упрощение и рефакторинг | Sonnet 4.5 |
| `onec-form-generator` | Генерация управляемых форм | Sonnet 4.5 |
| `onec-query-optimizer` | Оптимизация запросов | Sonnet 4.5 |
| `onec-test-generator` | Генерация тестов (Vanessa, YaXUnit) | Sonnet 4.5 |
| `onec-metadata-helper` | Навигация по метаданным | Sonnet 4.5 |
| `onec-admin` | Администрирование 1С серверов | Sonnet 4.5 |
| `mcp-deploy` | Управление MCP-серверами | Sonnet 4.5 |
| `dev-optimizer` | Мониторинг и оптимизация разработки | Sonnet 4.5 |

### Навыки (.cursor/skills/)

**Работа с конфигурацией:**
- `1c-batch` — выгрузка/загрузка конфигурации через ibcmd
- `1c-project-init` — инициализация нового проекта из базы

**Генерация объектов:**
- `1c-forms` — создание/редактирование управляемых форм
- `1c-mxl` — работа с макетами табличных документов
- `1c-roles` — создание ролей с правами доступа

**Интеграция с БСП:**
- `1c-bsp` — регистрация в подсистемах БСП

**Разработка:**
- `1c-feature-dev-enhanced` — полный цикл разработки функционала
- `1c-query-optimization` — продвинутые паттерны запросов

**Утилиты:**
- `auto-skill-bootstrap` — автоматическое обнаружение и установка навыков

### Правила (.cursor/rules/)

- `1c-coding-standards.mdc` — стандарты кодирования для 1С
- `onec.mdc` — специфика платформы 1С
- `sdd-workflow.mdc` — Specification-Driven Development
- `model-selection.mdc` — выбор модели AI (экономия токенов)
- `context-management.mdc` — управление контекстом и агентами
- `bsl-lsp-integration.mdc` — интеграция с BSL Language Server
- `mcp-tools-usage.mdc` — работа с MCP-инструментами
- `skills-first.mdc` — приоритет использования навыков
- `no-roi-estimates.mdc` — без оценок времени и ROI

| `1c-syntax-checker` | Проверка синтаксиса BSL | 💰 Платный |

**Примечание:** Платные MCP-серверы требуют подписки. Контакты для получения доступа: [ссылка будет добавлена]

#### Проектные MCP (создаются для вашего проекта)

Для каждого проекта 1С можно создать 2 специализированных MCP-сервера:

1. **Metadata & Code Search MCP** — семантический поиск по метаданным и коду
2. **Graph MCP** — граф зависимостей объектов конфигурации (Neo4j)

Индексация выполняется:
- В облаке (быстро, требует API ключ)
- Локально на Qwen 3 8B (бесплатно, медленнее)

Подробнее: [docs/guides/project-mcp-setup.md](docs/guides/project-mcp-setup.md)

## 🚀 Примеры использования

### Пример 1: Создание новой обработки с формой

```
Пользователь: Создай обработку "ЗагрузкаДанных" с формой для выбора файла

AI (использует агента onec-form-generator):
1. Создает метаданные обработки
2. Генерирует управляемую форму с полем выбора файла
3. Добавляет команду "Загрузить"
4. Валидирует через BSL LSP
5. Проверяет соответствие стандартам БСП
```

### Пример 2: Код-ревью модуля

```
Пользователь: Проверь модуль CommonModule.ОбщегоНазначения

AI (использует агента onec-code-reviewer):
1. Анализирует через BSL LSP (ошибки, предупреждения)
2. Проверяет соответствие стандартам кодирования
3. Ищет проблемы производительности
4. Проверяет использование БСП функций
5. Предлагает улучшения
```

### Пример 3: Оптимизация запроса

```
Пользователь: Оптимизируй этот запрос [код запроса]

AI (использует агента onec-query-optimizer):
1. Анализирует структуру запроса
2. Предлагает индексы
3. Переписывает с использованием временных таблиц
4. Проверяет через BSL LSP
5. Оценивает прирост производительности
```

## 💡 Best Practices

### Экономия токенов

- Используйте `rlm-toolkit` для сохранения контекста между чатами
- Запускайте агентов параллельно для изоляции контекста
- Начинайте новый чат при достижении 100k токенов
- Используйте Sonnet 4.5 для рутинных задач, Opus 4.6 только для критичных

### Качество кода

- Всегда используйте `onec-code-reviewer` перед коммитом
- Следуйте стандартам в `.cursor/rules/1c-coding-standards.mdc`
- Валидируйте формы через `1c-forms/validate`
- Проверяйте запросы через `onec-query-optimizer`

### Workflow

- Следуйте Specification-Driven Development (SDD)
- Используйте OpenSpec для документирования изменений
- Создавайте навыки для повторяющихся задач
- Используйте `dev-optimizer` для мониторинга эффективности
