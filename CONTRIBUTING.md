# Участие в разработке

Спасибо за интерес к проекту! Мы рады любому вкладу — от сообщений об ошибках до новых функций.

## Как помочь проекту?

### 🐛 Сообщить об ошибке

1. Проверьте, нет ли уже [похожей issue](https://github.com/Arman-Kudaibergenov/1c-ai-development-kit/issues)
2. Создайте новую issue с описанием:
   - Что вы делали
   - Что ожидали
   - Что получили
   - Версии (Cursor, 1С, ОС)
   - Логи (если есть)

### 💡 Предложить улучшение

1. Создайте issue с тегом `enhancement`
2. Опишите:
   - Проблему, которую решает улучшение
   - Предлагаемое решение
   - Альтернативы (если есть)

### 📝 Улучшить документацию

1. Найдите опечатку или неточность
2. Создайте PR с исправлением
3. Или создайте issue, если не уверены в правке

### 🔧 Создать новый навык или агента

1. Изучите существующие навыки в `.cursor/skills/`
2. Создайте новый навык по [шаблону](templates/skill-template.md)
3. Добавьте документацию
4. Создайте PR

### 🌐 Добавить перевод

1. Скопируйте `README.md` в `README.{lang}.md`
2. Переведите содержимое
3. Создайте PR

## Процесс разработки

### Настройка окружения

```bash
# Клонируйте репозиторий
git clone https://github.com/Arman-Kudaibergenov/1c-ai-development-kit.git
cd 1c-ai-development-kit

# Создайте ветку для работы
git checkout -b feature/my-awesome-feature
```

### Стандарты кодирования

#### Для навыков (SKILL.md)

- Используйте YAML блоки для структурированной информации
- Добавляйте примеры использования
- Документируйте все параметры
- Следуйте [шаблону навыка](templates/skill-template.md)

#### Для агентов (.md)

- Frontmatter с метаданными (name, description, model, priority)
- Четкое описание роли и ответственности
- Список доступных инструментов
- Примеры использования

#### Для правил (.mdc)

- YAML блоки для инструкций
- Примеры (хорошие и плохие)
- Интеграция с другими правилами
- Критические правила в конце

### Тестирование

Перед созданием PR:

1. **Проверьте навыки:**
   ```bash
   # Запустите навык вручную
   powershell -File .cursor/skills/your-skill/scripts/test.ps1
   ```

2. **Проверьте агентов:**
   - Откройте Cursor
   - Протестируйте агента на реальной задаче
   - Убедитесь, что он следует правилам

3. **Проверьте документацию:**
   - Все ссылки работают
   - Примеры корректны
   - Нет опечаток

### Создание Pull Request

1. **Коммиты:**
   ```bash
   git add .
   git commit -m "feat: add awesome feature"
   ```

   Используйте [Conventional Commits](https://www.conventionalcommits.org/):
   - `feat:` — новая функция
   - `fix:` — исправление ошибки
   - `docs:` — изменения в документации
   - `refactor:` — рефакторинг кода
   - `test:` — добавление тестов
   - `chore:` — изменения в сборке, CI и т.д.

2. **Push:**
   ```bash
   git push origin feature/my-awesome-feature
   ```

3. **Создайте PR:**
   - Опишите изменения
   - Добавьте скриншоты (если UI)
   - Укажите связанные issues

### Ревью кода

- Мы стараемся отвечать на PR в течение 48 часов
- Будьте готовы к правкам и обсуждению
- Все PR проходят автоматические проверки

## Стиль документации

### Markdown

- Используйте ATX заголовки (`#`, `##`, `###`)
- Код в блоках с указанием языка: ` ```bash`
- Списки: `-` для ненумерованных, `1.` для нумерованных
- Ссылки: `[текст](url)`

### Язык

- **Русский** — для пользовательской документации (README, гайды)
- **English** — для технической документации (агенты, правила, навыки)
- Причина: AI эффективнее обрабатывает английский в YAML/структурированном формате

### Примеры

Всегда добавляйте примеры:

```markdown
## Пример использования

### Задача
Создать обработку с формой

### Решение
\`\`\`
Пользователь: Создай обработку "ЗагрузкаДанных" с формой

AI: [использует навык 1c-forms/scaffold]
\`\`\`

### Результат
- Создана обработка в src/cf/DataProcessors/ЗагрузкаДанных/
- Создана форма Form.xml
- Добавлены метаданные
```
