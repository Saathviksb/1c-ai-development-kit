## 💡 Dmitrii Labintsev и RLM-toolkit

**Огромная благодарность [Dmitrii Labintsev](https://habr.com/ru/users/Dmitrii_Labintsev/)** за создание **RLM-toolkit** (Relational Long-term Memory):

### Что такое RLM-toolkit?

**RLM-toolkit** — это система иерархической памяти для AI-агентов, которая решает критическую проблему сохранения контекста между чатами.

### Вклад в проект

- 🧠 **Иерархическая память** — L0-L3 уровни для организации знаний
- 💾 **Персистентность** — сохранение контекста между сессиями
- 🔍 **Семантический поиск** — быстрый доступ к релевантной информации
- 🎯 **Каузальные связи** — запись решений с причинами и последствиями
- 🚀 **Экономия токенов** — 99.96% экономии на повторной загрузке контекста (50k → 440 токенов)
- 🆓 **Open Source** — бесплатный инструмент для всего сообщества

### Почему это важно?

Без RLM каждый новый чат начинается "с нуля". AI не помнит предыдущие решения, архитектурные выборы, стандарты проекта.

С RLM:
- AI помнит все важные решения
- Контекст загружается мгновенно
- Экономятся тысячи токенов
- Качество ответов выше (AI знает историю проекта)

### Использование в этом проекте

RLM-toolkit — один из двух **бесплатных** MCP-серверов в комплекте. Он используется:
- Для сохранения архитектурных решений
- Для хранения стандартов проекта
- Для передачи контекста между чатами
- Для оптимизации использования токенов

**RLM-toolkit сделал возможным эффективную работу с AI на больших проектах!**

## 🛠️ Инструменты и технологии

### Vladimir Akimov (SteelMorgan)

**Огромная благодарность [Vladimir Akimov](https://github.com/SteelMorgan)** за создание двух проектов, которые стали неотъемлемой частью этого инструментария:

#### mcp-bsl-lsp-bridge

[mcp-bsl-lsp-bridge](https://github.com/SteelMorgan/mcp-bsl-lsp-bridge) —

### Что такое mcp-bsl-lsp-bridge?

**mcp-bsl-lsp-bridge** — это MCP-сервер, который предоставляет AI-агентам доступ к возможностям BSL Language Server через единый интерфейс.

### Вклад в проект

- 🔌 **Singleton архитектура** — один сервер для всех агентов (экономия ресурсов)
- 🚀 **Нулевой overhead** — агенты подключаются мгновенно (без запуска BSL LS)
- 💾 **Shared cache** — диагностика, символы, hover информация кэшируются
- 🎯 **Экономия токенов** — 80% экономии на каждого агента (~800 токенов)
- 🆓 **Open Source** — бесплатный инструмент для всего сообщества

### Почему это важно?

Без mcp-bsl-lsp-bridge каждый AI-агент должен был бы:
- Запускать свой экземпляр BSL Language Server (~1000 токенов overhead)
- Тратить ресурсы на инициализацию (~5 секунд)
- Дублировать кэш диагностики и символов

С mcp-bsl-lsp-bridge:
- Все агенты используют один сервер
- Мгновенное подключение
- Общий кэш для всех
- Минимальное потребление ресурсов

### Использование в этом проекте

mcp-bsl-lsp-bridge — один из двух **бесплатных** MCP-серверов в комплекте. Он используется:
- Для диагностики кода (ошибки, предупреждения)
- Для навигации (go to definition, find references)
- Для рефакторинга (rename, code actions)
- Для форматирования кода

**mcp-bsl-lsp-bridge сделал возможным эффективную работу AI-агентов с BSL кодом!**

#### 1c-ai-sandbox-client-server

[1c-ai-sandbox-client-server](https://github.com/SteelMorgan/1c-ai-sandbox-client-server) — изолированная среда (Dev Container + Hyper-V VM) для безопасной работы AI-агентов с платформой 1С. Проект создан на личном опыте: автор потерял 10 лет семейных фотографий из-за неосторожных действий агента. Теперь агент работает в контейнере — хостовая система полностью защищена.
